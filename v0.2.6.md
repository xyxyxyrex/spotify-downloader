# Spoti-Tauri GUI Release v0.2.6

We are thrilled to roll out the **v0.2.6 release** of SpotDL GUI (Spoti-Tauri)! This release features a massive upgrade to our plugin system, bringing an official plug-and-play **Marketplace** catalog, robust **Uninstall/Delete** controls, a **Spotify Query Resolution Fix** for playlists, and built-in **Developer Tools** support for production profiles.

---

## 🚀 What's New in v0.2.6

### 🛒 Plugin Store Overhaul & Marketplace Integration
The plugin store has been completely redesigned for an intuitive, secure, and modern plug-and-play experience.
* **Separated Tabs & Statuses**: Installed plugins are now clean and isolated in the **Installed Plugins** panel, completely separated from the **Marketplace** catalog.
* **Live GitHub Marketplace Integration**: Discover, explore, and install community add-ons fetched dynamically from the official [spoti-tauri-plugin-marketplace](https://github.com/xyxyxyrex/spoti-tauri-plugin-marketplace).
* **Dynamically Loaded Modules**: Plugins are fetched from remote sources and securely loaded/instantiated dynamically via ES6 dynamic imports using safe `Blob` URLs.
* **Dynamic Uninstall / Deletion**: Uninstall any dynamic plugin with a simple button click. The UI removes them instantly with a smooth fade-out animation.
* **Removed Obsolete Local Loader**: The outdated and redundant local file-drop/code-paste loader has been removed to streamline the experience.

### 🔌 Official Marketplace Plugins
Two brand-new, robust plugins are now available in the store:
* **Receiptify**: Renders a gorgeous, custom receipt-style card summarizing your personal listening history and play counts, with an interactive modern interface.
* **History Stats**: Analyzes your local SQLite play history to generate dynamic, elegant statistical reports of your listening habits.

### 🛠️ Developer Tools Support
Debugging builds is now seamless:
* **DevTools Integration**: Enabled the native Safari/Chrome DevTools context menu in production and release compiles under debug profiles using the Tauri `devtools` cargo feature, letting you inspect components and view logs directly.

### 🐛 Critical Bug Fixes
* **Spotify Playlist Resolution Fix**: Resolved a critical issue in `spotify_query.py` where fetching playlists directly failed by bypassing unnecessary `init_spotify()` initialization checks.

---

## ⚙️ Compilation & Upgrade Path

All config manifests (`package.json`, `Cargo.toml`, `tauri.conf.json`, `updater.json`, and frontend elements) have been successfully bumped to `0.2.6`.

1. Run dependencies:
   ```bash
   npm install
   ```
2. Launch in development sandbox mode:
   ```bash
   npm run tauri dev
   ```
3. Compile production distributions (MSI, EXE, ZIP):
   ```bash
   npm run tauri build
   ```
